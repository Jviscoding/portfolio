01 / Perspective

Understanding the layers beneath the application interface.

I am naturally curious about the invisible parts of applications: the relational schema, the API boundary safety, and how data changes state dynamically.

I work across frontend and backend because modern systems are deeply interconnected. While I design fast React clients, my primary interest lies on the server side: designing stable REST APIs, structuring databases, handling secure authentication, and managing clean virtualization states.

I don't claim to be a veteran software engineer. Instead, I honestly communicate my process: I build practical systems to understand how they work, study engineering patterns systematically, and strive to make each new build more structured than the last.
