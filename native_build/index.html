01 / Perspective

Curious about how a product actually works.

I am drawn to the invisible parts of software: the data model behind a screen, the API boundary between services, the validation that protects a workflow, and the trade-offs that appear as an application grows.

I work across frontend and backend because useful systems need both. My strongest interest is backend development, where I can think through architecture, REST APIs, databases, authentication, performance, and the infrastructure that supports a reliable experience.

I do not claim to have all the answers or to be a software engineer already. I am continuously studying software engineering while building real systems, documenting what I learn, and making the next implementation more deliberate than the last.

01Inventory workflow

Stockora Inventory System

An evolving inventory workspace for managing products, brands, and dashboard information through defined frontend, API, and data boundaries.

ProblemKeep operational data consistent as product workflows grow beyond a single screen.
ApproachSeparate client features, authenticated API routes, and data access so each layer has a clear job.
DecisionsTypeScript feature modules, route separation, Supabase-backed auth, and Docker-based local workflows.
ReactTypeScriptExpressSupabasePostgreSQLDocker

The work is actively evolving. My focus is learning how choices such as API boundaries, idempotent operations, authentication checks, and data shape affect the usability and maintainability of an inventory product.

02Point of sale

PRISM POS & Inventory Manager

A point-of-sale and inventory project exploring how sales activity, products, user access, and reporting fit into one operational workflow.

ProblemConnect sales and stock updates without treating them as isolated records.
ApproachUse Firebase services for identity and synchronized data while keeping the project centered on workflow logic.
DecisionsManaged authentication and realtime data reduced setup overhead and surfaced business-rule decisions earlier.
